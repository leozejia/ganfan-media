# Claude Code 里跑 GPT，最怕的不是跑不起来

最怕的是它跑得起来。

做 SorryCode 中转的时候，我看到 sub2api 有个设置，可以让 GPT 跑在 Claude Code 里面。

第一眼看很酷。

Claude Code 的体验，GPT 的模型。

但这个组合有个问题：它太容易让人误会了。

用户会以为自己只是换了个模型。

实际换掉的是整条 agent 调用链路。

我见过一个用户跑了一天烧掉几十块，账单很吓人。

一问，原来是在用 Claude Code 跑 GPT。

产出却很迷惑：项目没推进多少，肉眼可见的输出也不多。

这时候骂模型贵，骂不到点上。

问题不是 GPT 贵。

问题是 GPT 被放进了一个不一定适合 GPT 的 agent runtime 里。

Claude Code 当然可以跑 GPT。

中转层把 Claude Code 的请求改写成 OpenAI 那边能理解的格式，再把结果包回来。

能回消息。

能执行任务。

看起来一切正常。

但 agent 的账单，不只看模型名字。

更要看一件事：

```text
同一批上下文，下一轮还能不能被便宜地复用
```

agent 不是每轮只把你刚说的话发出去。

它会把一整包上下文塞给模型：

```text
历史
文件
diff
工具
系统提示
压缩记录
```

如果这些稳定内容在请求开头保持一致，服务商就可能走缓存。

OpenAI 的 prompt caching 看的是精确前缀匹配。

前缀一样，才可能便宜。

Anthropic 也把缓存和工具、系统消息、messages 这类结构绑在一起。

换句话说，两家都不是只看“文字差不多”。

它们看的是请求结构稳不稳定。

所以 Claude Code 跑 GPT 的风险在这里：

Claude Code 原本按 Claude / Anthropic 的方式组织工具和上下文。

桥到 GPT 时，中间层要把这套结构翻译成 OpenAI 能理解的格式。

消息顺序、工具定义、系统提示、历史压缩，只要每轮稍微变一下，GPT 看到的就不是同一个稳定前缀。

那缓存命中就会变差。

缓存命中差，不是说模型更笨。

只是同一堆上下文，每轮都更像“重新读一遍”。

读得多，复用少，账单自然难看。

这不一定是中转层有恶意。

很多时候只是为了兼容，为了跑通。

但跑通和划算，是两回事。

所以那种“Claude Code 也能跑 GPT”的说法，缺了半句话：

```text
能跑。
但你要看 cached tokens。
```

如果 usage 里 cached tokens 长期很低，就说明大量重复上下文没吃上缓存。

这才是我看到那类账单时，最先怀疑的地方。

但别反过来迷信缓存。

缓存命中不是魔法。

官方文档里讲得很清楚：prompt caching 只是复用模型已经处理过的 prompt 前缀，降低延迟和成本。它不会改变模型最终生成什么。

换句话说，缓存不会让脏上下文变干净。

它只会让重复上下文变便宜。

重复上下文也分两种。

稳定的项目规则、工具定义、长期不变的背景，是资产。

已经失效的判断、走错方向的尝试、过期的错误信息、前几轮为了排障临时塞进去的上下文，是垃圾。

所以长会话不是越养越好。

一个会话养起来，可能发生两件事：

```text
稳定上下文被复用，成本下降
垃圾历史被保留，状态变脏
```

这就是双刃剑。

很多人只看到前半句。

长期跑代码任务，最理想的状态不是“会话越长越好”，而是：

```text
该稳定的东西稳定
该丢掉的东西丢掉
```

Codex 跑 GPT，是最合适的选择，好马配好鞍。

GPT 是它的主场。

这不是说 Codex 一定永远更便宜。

而是说，它至少少了一层协议翻译。

少一层翻译，就少一个把稳定前缀打散的地方。

所以 GPT 做任务，默认先用 Codex，自然是更合理的选择。

能跑，只说明桥通了。

不说明它省钱。

也不说明它适合长任务。

所以这篇只想讲一个很小的判断：

```text
GPT 做任务，默认先用 Codex。
Claude 做任务，默认先用 Claude Code。
```

OpenClaw、Hermes Agent、Claude Code 都可以通过自定义 endpoint 去跑 GPT。

可以玩，但那是进阶玩法。

账单变多，我会先查这里。

以后看到“某某 agent 也能跑 GPT”，先别兴奋。

先问两个问题：

```text
它能不能高效复用稳定上下文？
它会不会把垃圾历史一直带着跑？
```

这两个问题，比“能不能回消息”重要。

AI 工具现在最容易误导人的地方，就是把“能跑”包装成“适合”。

这两个词差得很远。

缓存命中也一样。

缓存命中高，不等于会话健康。

它只说明有一段东西被重复利用了。

至于那段东西是资产，还是垃圾，要你自己判断。
