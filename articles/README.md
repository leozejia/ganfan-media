# Articles

Each article or campaign gets its own folder:

```text
YYYY-MM-DD-topic-slug/
├── brief.md
├── sources.md
├── x.md
├── sorrycode.md
├── visual.md
├── assets/
│   ├── cover.png
│   └── inline-01.png
├── _work/
│   └── visual-runs/
├── outputs/
└── publish.md
```

Use `brief.md` to define the audience, angle, and factual boundary. Keep source
material in `sources.md`. Put final channel drafts in `x.md` and `sorrycode.md`.
Use `visual.md` as the visual decision source of truth. Put only selected
delivery assets in `assets/`. Put runtime attempts under `_work/visual-runs/`.
Put rendered channel artifacts under `outputs/`. Record shipped links and
feedback in `publish.md`.
