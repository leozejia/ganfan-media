# Visual Pattern Candidates

This is the internal incubation space for visual patterns that are useful in
GanFan production but not yet stable enough for `open-visual-grammar`.

Public Open Visual Grammar should only receive patterns with references,
boundaries, and production examples.

## Promotion rule

A candidate can move to `open-visual-grammar/patterns/` only when it has:

- a clear visual problem, not only a surface style;
- at least one public-safe reference image;
- use cases and non-use cases;
- anti-patterns;
- at least one useful production example;
- runtime adapter slots.

## Candidates

### Proof screenshot

Use when a reader needs real UI or command proof.

Rules:

- Use real screenshots or deterministic renders.
- Do not ask an image model to invent product UI.
- Crop, blur, or remove private data.
- Prefer one screenshot and one callout.
- Keep arrows and boxes minimal.

### Editorial illustration

Use when an essay or abstract narrative needs mood, metaphor, or human scale.

Rules:

- Use one metaphor and one emotional temperature.
- Avoid generated text unless typography is added manually later.
- Prefer tactile texture, human scale, and strong composition over generic AI symbols.
- The image should feel like an editorial argument, not a software banner.

### Minimal poster

Use when one object, phrase, or spatial relationship can carry the idea.

Rules:

- One object or one typographic phrase.
- Strong whitespace.
- Limited palette.
- No decorative UI cards.
- No dense explanatory text.

## Legacy surface styles

The old archived YAML files under `skills/ganfan-article-illustrator/styles/archive/`
are surface-style notes only. They are not patterns by themselves.

Treat them as production hints, not public grammar.
