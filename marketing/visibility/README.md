# Visibility

Updated: 2026-05-14

This is the operating home for SEO, GEO, and AI visibility work.

The goal is not to game ranking systems. The goal is to make real GanFan / SorryCode content easier for search engines and AI answer engines to discover, understand, cite, and describe accurately.

## Operating Position

- Build from real pages, real docs, and real product evidence.
- Prefer durable source-of-truth pages over short-term traffic tricks.
- Treat AI visibility as brand accuracy and discoverability, not as a standalone growth channel.
- Keep crawler, indexing, and project knowledge-page decisions visible to operators.
- Do not use cloaking, fake AI meta tags, hidden prompts, or SEO/GEO spam.

## Files

- `geo-ai-visibility.md` — strategy notes and implementation principles.
- `indexing-checklist.md` — recurring launch and maintenance checklist.
- `project-knowledge-page-template.md` — reusable structure for project/product knowledge pages.

## Cadence

Run the checklist when:

- a new public site, docs section, product page, or major article series goes live;
- a core SorryCode product page changes positioning;
- a project name, domain, canonical URL, or public docs path changes;
- monthly, for search index health and AI answer accuracy checks.
