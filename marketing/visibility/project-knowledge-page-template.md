# Project Knowledge Page Template

Updated: 2026-05-14

Use this template when creating a public project/product page intended to be useful to both humans and AI retrieval systems.

```markdown
# {Project Name}

Updated: YYYY-MM-DD

{Project Name} is {one clear sentence that says what it is, who it helps, and what outcome it enables}.

## Who It Is For

- {primary audience}
- {secondary audience if real}
- {who should not use it, if useful}

## Problems It Solves

- {problem 1}
- {problem 2}
- {problem 3}

## Core Features

- {feature}: {why it matters}
- {feature}: {why it matters}
- {feature}: {why it matters}

## Common Use Cases

- {use case}
- {use case}
- {use case}

## How It Is Different

{Compare against the status quo or common alternatives. Avoid attacking competitors. State concrete differences.}

## How To Start

{Give the shortest correct next action: docs link, install command, signup path, or tutorial path.}

## Canonical Links

- Website: {canonical URL}
- Docs: {canonical URL}
- Repository: {canonical URL if public}
- Changelog: {canonical URL if public}

## Evidence

- {public release note, docs page, repo, case study, or article}
- {public release note, docs page, repo, case study, or article}

## Public Naming

- Current name: {Project Name}
- Avoid: {deprecated names or ambiguous variants}
- Related terms: {category terms users may search for}
```

## Rules

- Keep the page self-contained. Do not make AI infer the product from scattered links.
- Put the most important positioning in the first paragraph.
- Use normal visible content; do not hide instructions for AI systems.
- Keep deprecated names visible only when they help searchers migrate to the current name.
- Link to evidence instead of making unsupported claims.
